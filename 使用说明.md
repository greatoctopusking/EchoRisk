## 1. 项目概述

EchoRisk 采用**两阶段训练策略**来解决小样本多视角超声心动图 EF 预测问题：

```
K400 预训练 → Phase 1: EchoNet-Dynamic 域内迁移 → encoder.pt
                                                ↓
                        Phase 2: CARDIOCARE 双视角微调 → best.pt
```

| 阶段 | 数据 | 样本数 | 目标 |
|---|---|---|---|
| K400 预训练 | Kinetics-400 动作识别 | ~240K | 学会通用视频理解 |
| **Phase 1** | EchoNet-Dynamic A4C 超声 | **7,460** | 从"看懂视频"→"看懂心脏超声" |
| **Phase 2** | CARDIOCARE A4C+A2C 双视角 | **495**（内部拆分） | 学跨视角融合 + 适配目标人群 |

### 核心特点

| 特性 | 说明 |
|---|---|
| 域内预训练 | EchoNet-Dynamic 超声→超声迁移，消除 K400 动作识别的域差距 |
| 双融合方案 | GatedFusion（2.5K 轻量）或 FusionBlock（token 级跨视角注意力） |
| 缺失鲁棒 | 可学习 NULL embedding 处理单视角缺失 |
| 患者级拆分 | 按 patient_id 划分训练/验证集，避免数据泄漏 |
| JSON 配置 | 所有参数支持 JSON 配置文件 |

## 2. 数据准备

### 2.1 CARDIOCARE 数据（Phase 2）

**DICOM 视频文件**，`pixel_array` shape 为 `(T, 708, 1016, 3)` 的三通道 RGB 图像。

目录结构：

```
{DICOM_ROOT}/
├── ECHORISK_0001/
│   ├── T1/
│   │   ├── ECHORISK_0001_T1_A4C.dcm
│   │   └── ECHORISK_0001_T1_A2C.dcm
│   ├── T2/
│   │   └── ...
│   └── ...
├── ECHORISK_0002/
│   └── ...
```

标签 CSV：

```csv
patient_id,timepoint,video_a4c,video_a2c,lvef,biomarker_elevated
ECHORISK_0001,T1,ECHORISK_0001_T1_A4C.dcm,,65.2,0
ECHORISK_0001,T2,ECHORISK_0001_T2_A4C.dcm,ECHORISK_0001_T2_A2C.dcm,62.1,0
```

- `video_a4c` / `video_a2c` 为空字符串表示该视角缺失
- `lvef` 为必填字段（EF 值，百分比）
- 使用 `utf-8-sig` 编码

### 2.2 EchoNet-Dynamic 数据（Phase 1）

公开数据集，`D:\echonetdynamic\`：

```
echonetdynamic/
├── FileList.csv          # 10,030 条：文件名、EF、ESV、EDV、FPS、帧数、Split
├── VolumeTracings.csv    # 左心室分割轮廓
└── Videos/
    ├── 0X100009310A3BD7FC.avi
    ├── ...
    └── (10,030 个 .avi 文件)
```

| 属性 | 值 |
|---|---|
| 视频格式 | `.avi`，3 通道 BGR，112×112 |
| 视角 | **仅 A4C**（无 A2C） |
| EF 范围 | 6.9 - 97.0，均值 55.7，std 12.4 |
| 帧数 | 28-1002 帧，均值 177 |
| Split | TRAIN=7,460 / VAL=1,288 / TEST=1,276 |

### 2.3 数据集划分

```
Phase 1 (EchoNet):
  FileList.csv 字段 "Split" = TRAIN / VAL / TEST

Phase 2 (CARDIOCARE):
  csv_train（全部训练数据）
  ├── 80% patients → 训练集 (split="train")
  └── 20% patients → 内部验证集 (split="val")
  csv_test → 留出测试集（可选，训练后评估用）
```

拆分**按 patient_id 确定性进行**（相同 seed 产生相同划分），确保同一患者的所有时间点（T1-T5）不会分散到训练集和验证集。

## 5. 快速开始

### 步骤 1：Phase 1 预训练

1. 编辑 `echonet_config.json`，设置 K400 权重路径：
```json
"model": {
    "pretrained": true,
    "weights": "D:/path/to/uniformer_small_k400_16x4.pth"
}
```

2. 运行预训练：
```bash
python train_echonet.py --config echonet_config.json
```
输出：`output/echonet_pretrain/encoder.pt`

### 步骤 2：Phase 2 微调

编辑 `train_config.json`，设置数据路径和 Phase 1 生成的 encoder 权重：

```bash
python main.py --config train_config.json
```

### 步骤 3：评估

```bash
python evaluate.py --config eval_config.json
```

## 6. 训练流程

### 6.1 Phase 1: EchoNet-Dynamic 预训练

**目标：** 将 K400 预训练的 UniFormer 迁移到超声域，学会从 A4C 超声视频中提取 EF 相关特征。

**配置：** `echonet_config.json`

| 参数 | 值 | 说明 |
|---|---|---|
| 数据集 | EchoNet-Dynamic A4C 单视角 | `EchoNet` Dataset 类 |
| 模型 | UniFormer-S → Linear(512,1) | 22M 参数 |
| 帧采样 | 36 帧，frequency=4 | 覆盖 4-5 个心动周期 |
| 分辨率 | 112×112 | 原生分辨率 |
| batch_size | 20 | |
| lr / wd | 1e-4 / 1e-4 | |
| epoch | 45，StepLR(step=15) | |
| 输出 | `encoder.pt`（去头，仅 backbone 权重） | |

**运行：**
```bash
python train_echonet.py --config echonet_config.json
```

**输出文件：**
```
output/echonet_pretrain/
├── log.csv          # 训练/验证日志
├── checkpoint.pt    # 最新 checkpoint
├── best.pt          # 验证集最优
└── encoder.pt       # 去除分类头的 encoder 权重（供 Phase 2 使用）
```

### 6.2 Phase 2: CARDIOCARE 微调

**目标：** 加载 Phase 1 的超声域 encoder，在 CARDIOCARE 双视角数据上微调融合层。

**配置：** `train_config.json`（GatedFusion）或 `train_config_fusionblock.json`（FusionBlock）

| 参数 | GatedFusion | FusionBlock | 说明 |
|---|---|---|---|
| frames / freq | 16 / 2 | 16 / 2 | CARDIOCARE 短视频适配 |
| resize | 112×112 | 112×112 | 与 Phase 1 对齐 |
| batch_size | 32 | 16 | FusionBlock 显存更大 |
| lr / wd | 5e-5 / 5e-4 | 5e-5 / 5e-4 | 小 lr 微调 |
| epochs / lr_step | 20 / 10 | 20 / 10 | |
| freeze_encoder_stages | 3（冻结 Stage 1-3） | 3 | 底层超声特征已固化 |
| modal_dropout | 0 | 0 | 训练信号最大化 |
| fusion_type | `"gated"` | `"fusionblock"` | 选融合方式 |
| echonet_weights | `output/echonet_pretrain/encoder.pt` | 同 | Phase 1 encoder 权重 |

**运行：**

```bash
# Exp A: GatedFusion 基线
python main.py --config train_config.json

# Exp B: FusionBlock 对照
python main.py --config train_config_fusionblock.json
```

**注意事项：**
- Phase 2 从头新建输出目录，不要和 Phase 1 混用
- 如果 `echonet_weights` 为空，则回退到 K400 权重（由 `pretrained` + `weights` 控制）
- 如果两者都为空，从头训练（不推荐）

### 6.3 Phase 3: 评估

```bash
python evaluate.py --config eval_config.json
```

评估输出：**eval_results/{exp_name}/**

详见 [第 10 章](#10-输出说明)。

## 9. 模型架构

### MultiModalEchoCoTr（Phase 2）

支持两种融合方式：

**GatedFusion (`fusion_type="gated"`，2.5K 参数)：**

```
A4C → Encoder → (B,512) ─┐
                           ├─ GatedFusion → Linear(512,1) → EF
A2C → Encoder → (B,512) ─┘
     ↑ 共享权重
```

- GatedFusion：`Linear(1024,2) → softmax → w₁·feat_A4C + w₂·feat_A2C`
- Encoder 加载 Phase 1 的 `encoder.pt`（超声域内权重）

**FusionBlock (`fusion_type="fusionblock"`，~1.2M 参数)：**

```
A4C → Encoder.forward_features → (B,512,8,3,3) ─┐
                                                  ├─ Concat → Conv3d proj → FusionBlock×1
A2C → Encoder.forward_features → (B,512,8,3,3) ─┘       │
     ↑ 共享权重                                      pool → MLP → EF
```

- 在特征图级别（非向量级别）融合，保留时空结构
- FusionBlock：Token 级跨视角自注意力（72 个时空 token）
- 精简设计：num_heads=4, mlp_ratio=1.0, drop_path=0.3

### UniFormer Encoder 结构

```
Stage 1: SpeicalPatchEmbed → CBlock×3   (64ch, T/2)
Stage 2: PatchEmbed        → CBlock×4   (128ch)
Stage 3: PatchEmbed        → SABlock×8  (320ch)    ← 冻结（freeze_encoder_stages≥3）
Stage 4: PatchEmbed        → SABlock×3  (512ch)    ← 可训练
         BN → 特征图 (B,512,T,H,W)
```

- Stage 1-2 用 CNN（`CBlock`）提取低级局部特征
- Stage 3-4 用 Transformer（`SABlock`）建模全局时空依赖
- 时间轴仅在入口处下采样一次（T→T/2），后续保持不变

### 缺失处理

- 单视角缺失时，缺失侧特征用**可学习 NULL embedding**（初始化为零）替代
- GatedFusion：缺失侧的门控权重硬置零并重新归一化
- FusionBlock：缺失侧特征图全部替换为 NULL embedding

## 11. 常见问题

### Q: Phase 1 预训练需要多久？

EchoNet-Dynamic 7,460 个视频，batch_size=20，36 帧，112×112，单 GPU A100 约 45 epoch × 每 epoch ~6 分钟 = 4-5 小时。

### Q: 训练时显存不足（OOM）

减小 `batch_size` 或 `resize`：

```bash
python main.py --config train_config.json --batch_size 8 --resize 112
```

### Q: 模型不收敛

1. 确认 Phase 1 的 `encoder.pt` 确实生成了且被正确加载
2. 检查 `echonet_weights` 路径是否正确（文件存在）
3. 将 `freeze_encoder_stages` 设为 0，让 encoder 也能微调
4. 检查 CARDIOCARE 数据像素值范围

### Q: 如何在已有 checkpoint 上恢复训练

直接运行相同的命令，脚本会自动检测 `checkpoint.pt` 并恢复。

### Q: echonet_weights 和 weights 有什么区别？

| 参数 | 加载目标 | 优先级 |
|---|---|---|
| `echonet_weights` | Phase 1 输出的 encoder.pt（超声域内权重） | **最高** |
| `weights` | K400 预训练权重（通用视频理解） | echonet_weights 为空时生效 |
| 都不设置 | 从头初始化 | 最低（不推荐） |

### Q: 如何只用 K400 预训练（跳过 Phase 1）

`train_config.json` 中设置：

```json
"pretrained": true,
"weights": "path/to/k400.pth",
"echonet_weights": null
```

### Q: 如何仅用单视角训练

Phase 2 中某个视角全部缺失时（CSV 中 `video_a2c` 全为空），模型会自动用 NULL embedding 处理。或设置 `--modal_dropout 1.0` 强制丢弃一个视角。

### Q: DICOM 预处理缓存

```bash
# 一次性预处理
python preprocess_cache.py \
    --csv_path "path/to/task1_train.csv" \
    --dicom_root "path/to/dicom/train" \
    --cache_dir "path/to/cache" \
    --resize 112

# 训练时启用
# train_config.json 的 "cache_dir": "path/to/cache"
```
