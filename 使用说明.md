## 1. 项目概述

EchoRisk 是一个基于深度学习的超声心动图（Echocardiogram）分析工具，用于从心脏超声视频中自动预测**左心室射血分数（LVEF）**。

### 核心特点

| 特性 | 说明 |
|---|---|
| 双视角融合 | 同时利用 A4C（心尖四腔）和 A2C（心尖两腔）视角 |
| 门控融合 | 可学习的 GatedFusion 自动分配视角权重 |
| 缺失鲁棒 | 支持单模态缺失，可学习 NULL embedding 兜底 |
| 模态 Dropout | 训练时随机丢弃一个视角（默认 15%），增强鲁棒性 |
| 患者级拆分 | 按 patient_id 划分训练/验证集，避免数据泄漏 |
| JSON 配置 | 命令行参数全支持 JSON 配置文件 |

### 工作流

```
┌─────────────┐     ┌──────────────┐     ┌──────────────┐
│ train_config │────▶│  main.py     │────▶│ checkpoint   │
│   .json      │     │  训练 + 测试  │     │ best.pt      │
└─────────────┘     └──────────────┘     └──────┬───────┘
                                                │
┌─────────────┐     ┌──────────────┐            │
│ eval_config │────▶│ evaluate.py  │◀───────────┘
│   .json      │     │  独立评估     │
└─────────────┘     └──────────────┘
```

## 2. 数据准备

### 2.1 数据格式

**DICOM 视频文件**，`pixel_array` shape 为 `(T, H, W, 3)` 的三通道 RGB 图像。

**目录结构：**

```
{DICOM_ROOT}/
├── ECHORISK_0001/
│   ├── T1/
│   │   ├── ECHORISK_0001_T1_A4C.dcm
│   │   └── ECHORISK_0001_T1_A2C.dcm
│   ├── T2/
│   │   ├── ECHORISK_0001_T2_A4C.dcm
│   │   └── ECHORISK_0001_T2_A2C.dcm
│   └── ...
├── ECHORISK_0002/
│   └── ...
└── ...
```

### 2.2 标签 CSV 格式

```csv
patient_id,timepoint,video_a4c,video_a2c,lvef,biomarker_elevated
ECHORISK_0001,T1,ECHORISK_0001_T1_A4C.dcm,,65.2,0
ECHORISK_0001,T2,ECHORISK_0001_T2_A4C.dcm,ECHORISK_0001_T2_A2C.dcm,62.1,0
```

- `video_a4c` / `video_a2c` 为空字符串表示该视角缺失
- `lvef` 为必填字段（EF 值，百分比）
- 使用 `utf-8-sig` 编码

### 2.3 数据集划分

数据通过 `train_config.json` 中的配置进行划分：

```
csv_train (所有训练数据)
├── 80% patients → 训练集 (split="train")
└── 20% patients → 内部验证集 (split="val")

csv_test → 留出测试集 (split="test")
```

拆分**按 patient_id 确定性进行**（相同 seed 产生相同划分），确保同一患者的所有时间点（T1-T5）不会分散到训练集和验证集。

## 5. 快速开始

### 步骤 1：修改配置文件

编辑 `train_config.json`，设置数据路径：

```json
{
    "data": {
        "csv_train": "D:/your/path/Task1/labels/task1_train.csv",
        "csv_test": null,
        "dicom_root": "D:/your/path/Task1/dicom/train"
    }
}
```

### 步骤 2：训练

```bash
python main.py --config train_config.json
```

仅覆盖个别参数：

```bash
python main.py --config train_config.json --epochs 100 --batch_size 4
```

### 步骤 3：评估

编辑 `eval_config.json`，设置 `checkpoint` 路径为训练生成的 `best.pt`，然后运行：

```bash
python evaluate.py --config eval_config.json
```

## 8. 模型架构

```
输入: A4C视频 (B,3,T,224,224)    A2C视频 (B,3,T,224,224)
         │                              │
         ▼                              ▼
    UniFormer Encoder              UniFormer Encoder       ← 共享权重
         │                              │
         ▼                              ▼
     feat_A4C (B,512)              feat_A2C (B,512)
         │                              │
         └──────────┬───────────────────┘
                    ▼
              GatedFusion
        Linear(1024,2) → softmax
        w_A4C·feat_A4C + w_A2C·feat_A2C
                    │
                    ▼
            Linear(512,1) → EF 预测值
```

### UniFormer 结构

```
Stage 1: SpeicalPatchEmbed → CBlock×3   (64ch, T/2)
Stage 2: PatchEmbed        → CBlock×4   (128ch)
Stage 3: PatchEmbed        → SABlock×8  (320ch)
Stage 4: PatchEmbed        → SABlock×3  (512ch)
         BN → GlobalAvgPool → 512d 特征向量
```

- Stage 1-2 用 CNN（`CBlock`）提取低级局部特征
- Stage 3-4 用 Transformer（`SABlock`）建模全局时空依赖
- 时间轴仅在入口处下采样一次（T→T/2），后续保持不变

### 缺失处理

- 单视角缺失时，缺失侧特征用**可学习 NULL embedding**（初始化为零向量）替代
- GatedFusion 将缺失侧的门控权重**硬置零**，并重新归一化另一侧的权重

## 10. 常见问题

### Q: 训练时显存不足（OOM）

减小 `batch_size`（如 `4` 或 `2`）或缩小 `resize`（如 `160`）：

```bash
python main.py --config train_config.json --batch_size 2 --resize 160
```

### Q: 模型不收敛

1. 检查 `lr`，尝试从更小的学习率开始（如 `5e-5`）
2. 确保 `modal_dropout` 不要设太大（建议 0-0.3）
3. 检查数据均值/标准差是否在合理范围
4. 检查 DICOM 像素值范围（应为 0-255 uint8）

### Q: 如何在已有 checkpoint 上恢复训练

直接运行相同的命令即可，脚本会自动检测 `checkpoint.pt` 并恢复：

```bash
python main.py --config train_config.json
```

### Q: 评估时需要 mean/std

如果不提供，评估脚本会使用 `mean=0, std=1`。若要复现训练时的设定，有两种方式：

1. 命令行传入（逗号分隔）：
```bash
python evaluate.py --config eval_config.json \
    --mean "0.485,0.456,0.406" --std "0.229,0.224,0.225"
```

2. 在 `eval_config.json` 中设置 `"mean"` 和 `"std"` 为训练日志中打印的值。

### Q: 如何仅用单视角训练

设置 `--modal_dropout 1.0 --train_split_ratio 1.0`（无验证集），模型会自动用 NULL embedding 替代缺失视角。

### Q: csv_train 和 csv_test 是同一批数据的不同拆分吗

不同。`csv_train` 用于训练和内部验证（通过 `train_split_ratio` 按 patient 拆分），`csv_test` 是完全独立的留出测试集。通常 `csv_train` = 竞赛训练集，`csv_test` = 竞赛验证集。

### Q: pretrained 权重从哪里来

UniFormer 的 Kinetics-400 预训练权重：https://huggingface.co/Sense-X/uniformer_video

下载后用 `--weights` 参数指向 `.pth` 文件路径。
